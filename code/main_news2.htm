There are many bugs I found in Apollo v0.01d R6-RC1 which went out to Emulation64 staff and friends. I think a rewrite is due. For starters, I HATE the plugin selection dialog box, the multilingual support, and all the "dynarec" options in the GUI that Phrodide put in there. Also, the timing of the CPU is horrid, the interrupt handling leaves something to be desired, and I would rather rework a few things in the CPU, especially the opcode fetch/translation loop. The method I have done now is VERY fast, but it could be faster by removing unneeded code from the loop and removing stack operations (thus using naked functions). This will also come in handy when I do my psuedo dynarec engine which will make code to call each function individually (rather then recompilation). That is more like a caching interpretive and it removes all need to fetch/translate, but it doesn't remove function calling or allow for need-based optimizations. Probably more technical talk then most of you care to hear, but it's the next step to producing a dynarec without leaving the interpretive stages completely yet. All status updates from here on out will be in the developer's log save for a few points of interest I will place right on the main page.
