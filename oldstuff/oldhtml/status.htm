Status

 

Current Status of the Apollo Project (24.June.2000):


Main Processor and co-processors:

  • R4300i - No implementation of trap instructions, LoadLinked, or StoreConditional opcodes.
  • MMU - No emulation of the Watch Registers... Possibly improper emulation of the count register (46Mhz vs 92Mhz).
  • FPU - Inaccurate emulation of FC31 except for rounding mode, no initial value of FC0.

    • Memories:

  • RDRAM (main memory) Emulating 8MB.
  • MI (MIPS Interface) MI[0] emulated partially, all others correctly emulated.
  • VI (Video Interface) Only 16bit mode supported, most registers ignored.
  • AI (Audio Interface) Almost put in correctly. Dummy support causes some inaccurate emulation. 
  • PI (Parallel Interface) DMA from ROM->RDRAM supported. RDRAM->ROM not supported.
  • SI (Serial Interface) Correctly Emulated.
  • SP (Signal Processor) Registor support in excluding the semaphore. See below for processor info.
  • DP (Display Processor) Register support included. See below for processor info.
  • PIF (Controllers) Commands 0 correctly emulated. Adaptoid Support has been put in which emulates the command 01 also. (01 is the main command)
  • TLB. This section is being reworked for better performance and easier to read code.
    •

    Interrupts/Exceptions:

  • Exceptions are not handled correctly and are ignored. This will change soon with the dynarec core put in place.
  • Count Interrupts (the N64 timer) As stated above, this can possibly be wrong. Need to verify.
  • Vsync Interrupts - Known to be inconsistent with the actual workings of the VI hardware. Will rework this interrupt when new interrupt system is put into place.
  • Serial Interface Interrupts - Working good.
  • Parallel Interface Interupts - Working good.
  • Audio Interface Interupts - Working good.
  • Signal Processor Interrupts - Working good.
  • Display Processor Interrupts - Working good.
    •

    RSP/RDP (graphics emulation):

  • Dummy support has been put in for the RSP and RDP. Expect more complete support later on.
